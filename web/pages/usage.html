概述

TsToHls 是一个将 M3U 播放列表转换为 HLS 格式的工具，支持将直播源转换为可在浏览器和各种播放器中播放的 HLS 流。

v1.4.1 采用进程内切片架构（go-astiav），替代了外部 FFmpeg 进程调用，实现首帧快速启动、SegmentStore 内存直出零磁盘 IO、音频实时转码（mp2 → aac）。

系统安装

使用 Docker Compose 来部署 TsToHls，这样可以更方便地管理依赖和环境。

1. 创建 docker-compose.yml 文件

在您的项目目录中创建一个 docker-compose.yml 文件，内容如下：

services:
  tstohls:
    image: ghcr.io/kronus09/tstohls:latest
    container_name: tstohls
    restart: unless-stopped
    ports:
      - "15140:15140"
    volumes:
      # 数据持久化目录（数据库、配置、M3U文件、图标）
      - ./data:/app/data
    environment:
      - TZ=Asia/Shanghai
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

2. 启动服务

docker-compose up -d

3. 访问管理界面

服务启动后，您可以在浏览器中访问 http://localhost:15140 打开管理界面。

注意： 使用 Docker 部署时，需要确保您的系统已经安装了 Docker 和 Docker Compose。

快速开始

服务启动后，您可以通过以下地址访问 TsToHls：

管理界面： http://[服务器IP]:15140

订阅地址： http://[服务器IP]:15140/playlist/tstohls.m3u

功能说明

数据导入

支持本地 M3U 文件上传和远程订阅地址导入，自动解析并转换为 HLS 格式。

提供「验证源可靠性」开关：开启时探测源编码，H.264 立即可用，非 H.264 后台异步补探；关闭时也探测但全部启用。

系统监控

管理界面顶部显示：活跃切片路数、频道可用/总数（绿色/灰色）、CPU 占用、内存使用。

切片器配置

点击「编辑配置」按钮，可调整切片器参数。建议保持默认配置。

订阅地址

生成的 HLS 订阅地址可在 PotPlayer、VLC、IINA、nPlayer 等支持 HLS 的播放器中使用。

原流播放

实验性功能，允许直接在浏览器中播放原始 M3U 源中的 TS 流，无需转码切片。

功能特点

  • 更低延迟：直接播放原始 TS 流，省去转码和切片时间
  • 源对照：可用于验证原始源是否可用，排查转换问题

使用方式

点击顶部选项卡的「原流播放」即可进入原流播放页面。

技术原理

使用 mpegts.js 库在浏览器中直接解析和播放 MPEG-TS 流，通过 Media Source Extensions (MSE) 技术实现。

兼容性提示： 此功能未经转码，仅PC端浏览器支持。移动设备请使用 HLS 播放。

常见问题

1. 导入失败怎么办？

请检查 M3U 文件格式是否正确，或订阅地址是否可访问。

2. 播放时出现卡顿怎么办？

可以尝试调整切片器配置中的切片时长和列表容量，或增加服务器网络带宽。

3. 如何更新源？

直接点击「上传并转换」按钮，系统会从订阅地址重新拉取最新的源并重新探测。

4. 支持哪些播放器？

  • 桌面端：PotPlayer、VLC、IINA、MPV
  • 移动端：nPlayer、Fileball、APM、Kodi
  • 浏览器：Chrome、Firefox、Safari

切片器配置

v1.4.1 使用进程内切片架构，配置项如下：

  • 最大并发：同时切片的最大路数（默认 6）
  • 切片时长：每个 TS 分片的时长，单位秒（默认 1）
  • 列表容量：M3U8 中保留的切片数（默认 4）
  • 闲置超时：无访问后自动释放切片器的秒数（默认 120）
  • 重连延迟：断流后重试间隔秒数（默认 5）
  • 音频编码器：aac（推荐，实时转码 mp2→aac）或 copy（直接复制）
  • 音频码流：AAC 编码码率，可选 64k / 96k / 128k / 192k / 256k
架构说明： 切片器在进程内运行（go-astiav + CGO），视频流直接 copy，音频实时转码为 AAC。HLS 切片数据存储在内存中（SegmentStore），HTTP 请求直接读取内存返回，零磁盘 IO。从第一帧直接开始输出，切分边界在关键帧处确保播放器正确解码。
