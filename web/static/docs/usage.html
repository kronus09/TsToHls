概述

TsToHls 是一个将 M3U 播放列表转换为 HLS 格式的工具，支持将直播源转换为可在浏览器和各种播放器中播放的 HLS 流。

系统安装

使用 Docker Compose 来部署 TsToHls，这样可以更方便地管理依赖和环境。

1. 创建 docker-compose.yml 文件

在您的项目目录中创建一个 docker-compose.yml 文件，内容如下：

services:
  tstohls:
    image: ghcr.io/kronus09/tstohls:latest
    container_name: tstohls
    restart: unless-stopped
    ports:
      - "15140:15140"
    volumes:
      # 映射生成的 mapping.json 和新 m3u 所在的目录到宿主机
      # 这样你重启容器后，之前验证过的频道信息不会丢失，不用重新探测
      - ./m3u:/app/m3u
      # 如果你需要手动上传原始 iptv.m3u 到容器，也可以映射整个根目录或特定文件
      # - ./iptv.m3u:/app/iptv.m3u
    tmpfs:
      # 将新的切片目录 hls_temp 挂载到内存中
      # size=512M 足以支撑 10-20 个频道同时点播（每个频道切片约占用 20-30MB）
      - /app/hls_temp:size=512M,mode=1777,exec
    environment:
      - GIN_MODE=release
      - TZ=Asia/Shanghai
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

2. 启动服务

在包含 docker-compose.yml 文件的目录中执行以下命令：

docker-compose up -d

3. 访问管理界面

服务启动后，您可以在浏览器中访问 http://localhost:15140 打开管理界面。

注意： 使用 Docker 部署时，需要确保您的系统已经安装了 Docker 和 Docker Compose。

快速开始

服务启动后，您可以通过以下地址访问 TsToHls：

管理界面： http://[Docker服务器IP]:15140

订阅地址： http://[Docker服务器IP]:15140/playlist/tstohls.m3u

功能说明

数据导入

支持本地 M3U 文件上传和远程订阅地址导入，自动解析并转换为 HLS 格式。

提供「仅保留 H.264 格式」选项，可过滤非 H.264 编码的视频流，提高播放兼容性。

系统监控

在管理界面顶部，您可以查看系统的活跃进程数、频道总数、CPU 占用和内存使用情况。

FFMPEG 配置

点击「编辑配置」按钮，您可以调整 FFMPEG 的各种参数，如最大并发进程数、切片时长、列表容量等。

建议保持默认配置，除非您对 FFMPEG 有深入了解。

订阅地址

生成的 HLS 订阅地址可以在各种支持 HLS 格式的播放器中使用，如 PotPlayer、VLC、IINA 等。

移动设备上可以使用 nPlayer、Fileball、APM 等播放器。

实验功能

Direct TS 直接播放

Direct TS 是一个实验性功能，允许直接在浏览器中播放原始 M3U 源中的 TS 流，无需经过 FFmpeg 转换为 HLS 格式。

功能特点

  • 更低延迟：直接播放原始 TS 流，省去了转码和切片的时间
  • 更快启动：无需等待 FFmpeg 初始化，点击即播
  • 源对照：可用于验证原始源是否可用，排查转换问题

使用方式

在管理界面的「订阅地址」板块底部，点击「Direct TS 实验播放」区域的「点击进入」按钮即可进入实验播放页面。

技术原理

该功能使用 mpegts.js 库在浏览器中直接解析和播放 MPEG-TS 流，通过 Media Source Extensions (MSE) 技术实现。

注意： 此功能为实验性质，可能存在兼容性问题。部分浏览器或设备可能无法正常播放。如遇到问题，请使用标准的 HLS 订阅地址。
兼容性提示： 需要浏览器支持 MSE (Media Source Extensions) 技术。Chrome、Firefox、Edge 等现代浏览器通常支持，但部分移动浏览器可能不支持。

常见问题

1. 转换失败怎么办？

请检查 M3U 文件格式是否正确，或订阅地址是否可访问。如果问题持续存在，请查看终端中的错误信息。

2. 播放时出现卡顿怎么办？

可以尝试调整 FFMPEG 配置中的切片时长和列表容量，或增加服务器的网络带宽和硬件资源。

3. 如何更新源？

如果您是通过订阅地址导入的，可以直接点击「上传并转换」按钮，系统会从订阅地址重新拉取最新的源。

如果您是通过本地文件导入的，可以重新上传更新后的 M3U 文件。

4. 支持哪些播放器？

支持所有支持 HLS 格式的播放器，包括但不限于：

  • 桌面端：PotPlayer、VLC、IINA、MPV
  • 移动端：nPlayer、Fileball、APM、Kodi
  • 浏览器：支持 HLS 的现代浏览器（如 Chrome、Firefox、Safari）

高级功能

专家配置

在「FFMPEG 专家配置」部分，您可以调整以下参数：

  • 最大并发进程：控制同时转换的频道数量
  • 切片时长：控制 HLS 切片的长度
  • 列表容量：控制 M3U8 文件中包含的切片数量
  • 闲置超时：控制空闲频道的自动释放时间
  • 视频/音频编码器：选择视频和音频的编码方式
注意： 调整这些参数可能会影响系统性能和播放质量，请根据实际情况进行调整。
