# TsToHls - 直播流 HLS 切片工具

将 TS 协议直播流转换为 HLS 格式的工具，让家里的 IPTV 可在浏览器内播放，支持输出 M3U 频道订阅。

## v1.4.2 更新亮点

> 从 v1.3.x 升级时，系统会**自动迁移历史数据**（源文件、图标），无需手动操作。

- **播放秒开**：进程内切片架构（go-astiav + CGO）替代外部 FFmpeg 进程，省去进程启动开销；从第一帧直接输出，不再等待关键帧对齐
- **内存直出**：SegmentStore 内存存储 HLS 切片，HTTP 直接读取内存返回，零磁盘 IO，消除文件读写延迟
- **音频实时转码**：mp2 → aac 进程内实时转码（decode → filter → encode），浏览器原生可播
- **后台补探**：H.264 频道同步探测立即可用，非 H.264 异步后台补探，导入速度提升数倍
- **SQLite 持久化**：频道元数据、探测结果、启禁状态全部存库，重启不丢失
- **自动迁移**：从 v1.3.x 升级时自动转换历史数据，升级无感知

## 特性

- 实时切片 TS 流为 HLS（H.264 视频 copy，音频 mp2→aac 实时转码）
- 进程内切片架构，首帧快速启动，SegmentStore 内存直出零磁盘 IO
- **原流播放**：直接在浏览器中播放原始 TS 流，更低延迟（仅 PC 端支持）
- M3U/M3U8 文件管理，SQLite + WAL 持久化频道元数据
- 轻量高效，支持容器化部署
- 简易 Web 管理界面，可用页面直接播放预览
- 分组频道显示，可用/总数统计
- 切片器配置，可调整切片和转码参数

## 快速开始

### Docker Compose 部署

1. 创建 `docker-compose.yml` 文件：

```yaml
services:
  tstohls:
    image: ghcr.io/kronus09/tstohls:latest
    container_name: tstohls
    restart: unless-stopped
    ports:
      - "15140:15140"
    volumes:
      # 数据持久化目录（数据库、配置、M3U文件、图标）
      - ./data:/app/data
    environment:
      - TZ=Asia/Shanghai
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
```

2. 启动服务：
```bash
docker-compose up -d
```

3. 访问管理界面：
```
http://服务器IP:15140
```

### 从 v1.3.x 升级

直接替换 Docker 镜像重启即可，系统自动完成：
- 迁移 `data/m3u/` 下的源文件和图标到 `data/` 平铺位置
- 删除旧格式配置文件，自动创建新格式
- 重新导入频道数据到 SQLite 数据库
- 保留已下载的频道图标

升级过程中前端会显示「系统升级中」提示，等待完成后自动恢复。

## 功能说明

### 数据导入
- 支持本地 M3U 文件上传
- 支持远程订阅地址导入
- 「验证源可靠性」开关：开启时 H.264 立即可用，非 H.264 后台异步补探；关闭时也探测但全部启用

### HLS 播放
- 进程内切片，首帧快速启动
- 视频流 copy，音频 mp2→aac 实时转码
- SegmentStore 内存直出，零磁盘 IO
- 按分组显示频道列表，分组按钮显示频道数量统计

### 原流播放
- **更低延迟**：直接播放原始 TS 流，省去转码时间
- **源对照**：可用于验证原始源是否可用
- **兼容性**：仅 PC 端浏览器支持，移动设备请使用 HLS 播放

### 切片器配置
- 最大并发路数（默认 6）
- 切片时长（默认 1 秒）
- 列表容量（默认 4 个切片）
- 闲置超时（默认 120 秒）
- 重连延迟（默认 5 秒）
- 音频编码器：aac（推荐）或 copy
- 音频码流：64k / 96k / 128k / 192k / 256k

## 配置说明

- **数据持久化**：`./data` 目录存储 SQLite 数据库、配置文件、M3U 源文件、本地化图标
- **时区**：默认 `Asia/Shanghai`，可按需修改

## 系统要求

- Docker 环境
- 推荐 2GB 以上内存（支持更多并发频道）
- 稳定的网络连接

## 常见问题

**Q: 导入失败怎么办？**
A: 请检查 M3U 文件格式是否正确，或订阅地址是否可访问。

**Q: 播放时出现卡顿怎么办？**
A: 可以尝试调整切片器配置中的切片时长和列表容量，或增加服务器网络带宽。

**Q: 如何更新源？**
A: 直接点击「上传并转换」按钮，系统会从订阅地址重新拉取最新的源并重新探测。

**Q: 支持哪些播放器？**
A: 支持所有支持 HLS 格式的播放器，包括但不限于：
- 桌面端：PotPlayer、VLC、IINA、MPV
- 移动端：nPlayer、Fileball、APM、Kodi
- 浏览器：Chrome、Firefox、Safari

**Q: 从 v1.3.x 升级需要注意什么？**
A: 直接替换镜像重启即可，系统自动迁移。升级期间页面会显示提示，等待完成后自动恢复。旧的 `hls_temp/` 和 `mapping.json` 不再使用，会自动清理。
